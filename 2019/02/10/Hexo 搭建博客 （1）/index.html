Hexo 是什么？

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

详情请查阅官网 >> [GO]

Hexo 能干什么？

GitHub Pages + Hexo 搭建博客站点

Coding Pages + Hexo 搭建博客站点

Heroku + Hexo 搭建博客站点

1. Hexo 本地安装

实例环境： Win 10 专业版 64位操作系统

这里仅提供一种使用方案

1、 >> 安装前提： 在安装前，必须检查本地电脑中是否已安装下列应用程序：

  • Node.js
  • Git

如果没有安装请先安装

请按自身操作系统类型选择 32-bit 版本（基于X86处理器） 或 64-bit 版本（基于X64处理器）

  • Node.js >> [GET]
    推荐下载LTS（长期支持版本）

  • Git >> [GET]

一路下一步安装好两个应用程序即可

注：如果遇到意外报错，请重新安装node.js程序。

如果你的电脑中已经安装上述必备程序，那么接下来只需要使用 npm 即可完成 Hexo 的安装。

2、 >> 右键菜单栏里打开 Git Bash 输入命令回车

$ npm install -g hexo-cli

3、 >> 安装 Hexo 完成后，请依次执行下列命令，Hexo 将会在指定文件夹中新建所需要的文件。

hexo init blog 指的是在当前文件夹下新建blog目录作为博客的根目录

$ cd ~
$ hexo init <folder> #请自定义博客根目录，这里用 blog 替换 <folder>
$ cd <folder>
$ npm install #这是安装所有的依赖包

新建完成后，指定文件夹的目录如下：

.
├── _config.yml #网站的配置信息，可以在此配置大部分的参数。
├── package.json #应用程序的信息
├── scaffolds #模版文件夹
├── source #资源文件夹是存放用户资源的地方
|   ├── _drafts #草稿
|   └── _posts #文章
└── themes #主题文件夹 Hexo 会根据主题来生成静态页面

2. NexT 主题安装及基础配置

这里以使用 NexT 主题 为例

NexT 主题官网 >> [GO]

1、 >> 下载主题

打开 Git Bash 将主题文件拷贝至站点目录的 themes 目录下

$ cd your-hexo-site #例如blog目录（cd ~/blog）
$ git clone https://github.com/iissnan/hexo-theme-next themes/next

2、>> 启用主题

与所有 Hexo 主题启用的模式一样。 当 克隆/下载 完成后，打开 站点配置文件， 找到 theme 字段，并将其值更改为 next。

站点配置文件 （~/blog/_config.yml）（ ~ 是指用户目录如「C:\Users\Administrator」）

到此，已经成功安装并启用了 NexT 主题。

!!! 每次配置完成后请立即 Ctrl+S 保存更改

3、>> 主题设定

主题配置文件，搜索（Ctrl+F） scheme 关键字。 你会看到有四行 scheme 的配置，将你需用启用的 scheme 前面注释 # 去除即可。

主题配置文件（~/blog/themes/next/_config.yml）（~ 指用户目录如「C:\Users\Administrator」）

#scheme: Muse
#scheme: Mist
#scheme: Pisces
scheme: Gemini

4、>> 设置 语言

编辑 站点配置文件， 将 language 设置成你所需要的语言。这里选用简体中文，配置如下：

language: zh-Hans

3. 本地预览

开始前请确保 Hexo 和 NexT 主题已经正常安装

hexo g # 等同于 hexo generate，生成静态文件
hexo s # 等同于 hexo server，在本地服务器运行

打开浏览器输入 http://localhost:4000 能够访问说明部署成功。

网站配置信息

你可以在 _config.yml （站点配置文件）中修改大部分的配置。
